自定义jQuery插件

什么是插件?

就是一些封装好的jquery方法,通过调用插件写好的方法,快捷实现某些功能,而不用自己再用代码实现.

jQuery插件分为两种:

  • 类插件 (为jQuery添加新的全局函数,为jQuery类添加新的方法) $.fn({})
  • 对象插件 (给jQuery对象添加方法) $("str").fn({})

jQuery插件开发要点(参考)

  • 插件的文件命名必须严格遵循jQuery的js的规则,以便于与其他JS文件的区分,如新插件文件jquery.newplugin.js
  • 如果是对象级别插件,所有的方法都应依附于jquery.fn主体对象,如果是类级别插件,所有的方法都应依附于jquery对象;
  • 无论是对象级还是类级别插件,结尾都必须以分号结束,否则,在文件被压缩时,会出现错误提示信息;
  • 在插件内部的代码中,如果要访问每个元素,可以使用this.each方法来遍历全部元素.
  • 需要说明的是插件的内部,this所代表的是通过jQuery选择器所获取的对象,而非传统意义上的对象的引用;
  • 由于jQuery代码在调用方法时,可以采用链写的方法同时调用多个方法,因此,为了保证这个功能的实现,插件本身必须返回一个jQuery对象.
  • 虽然"$"美元符,可以与"jquery"字符相代替,但在编写插件的代码中,尽量不要使用"$"符号,以避免与别的代码冲突.
  • 在编写对象级别的插件时,使用jQuery.fx.entend()方法进行功能扩展,而针对类级别的插件,则使用jQuery.extend()方法进行扩展.

开发对象级别插件

形式a

(function($){ $.fn.extend({ pluginName:function(opt,callback){ } }) })(jQuery);

形式b

(function($){ $.fn.pluginName = function(opt,callback) { } })(jQuery);

类对象级别插件

形式a

jQuery.extend({ pluginName:function(){ } })

形式a

(function($){ $.extend({ pluginName:function(opt,callback){ } }) })(jQuery);
